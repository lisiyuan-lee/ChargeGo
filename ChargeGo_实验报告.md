# 《智能充电桩调度计费系统》开发实验报告

> **项目名称**: ChargeGo — 智能充电桩调度计费系统  
> **开发周期**: 2026年春季学期  
> **报告日期**: 2026-06-17  

---

## 一、实验目的

1. 设计并实现一个智能充电桩调度与计费管理系统，支持电动汽车用户的充电申请、排队调度、实时计费等功能。
2. 采用前后端分离架构，后端基于 FastAPI + SQLAlchemy，前端基于 Vue 3 + Element Plus，构建完整的 Web 应用。
3. 实现分时电价计费、多策略故障调度、暂停超时罚金等业务规则。
4. 通过系统化的验收测试，验证系统功能完整性、配置准确性、计费正确性和多车辆并发处理能力。

### 3.2 数据库设计

系统共包含 **4 张核心数据表**：

| 表名 | 主要字段 | 说明 |
|------|---------|------|
| **users** | id, username, password_hash, role, nickname, car_id, battery_capacity, warning_count | 用户账户信息，区分 user/admin 角色 |
| **charging_piles** | id, code, type, power, is_working | 充电桩配置，支持动态调整功率 |
| **charge_requests** | id, user_id, pile_id, mode, requested_energy, status, queue_number, paused_energy, total_paused_minutes | 全生命周期充电请求记录 |
| **charge_bills** | id, request_id, user_id, pile_id, charge_energy, electricity_fee, service_fee, total_fee, notes | 计费账单，含罚金备注 |

**状态机设计**（充电请求状态流转）：

```
waiting ──→ queued ──→ charging ──→ completed
   │                    │  │
   └──→ cancelled      │  └──→ paused ──→ charging (恢复)
                        │         │
                        └──→ fault_waiting ──→ queued (故障恢复)
```

## 四、关键功能实现

### 4.1 分时电价计费

系统实现了**峰平谷三时段电价**，`electricity_rate()` 函数根据当前小时返回对应电价：

$$
\text{总费用} = \text{充电电量} \times \text{电价}(t) + \text{充电电量} \times \text{服务费率}
$$

| 时段 | 时间范围 | 电价 | 服务费 | 总费率 |
|:----:|:--------:|:----:|:-----:|:------:|
| 🌙 谷时 | 00:00-07:00, 23:00-24:00 | 0.4 | 0.8 | **1.2** |
| 🌤️ 平时 | 07:00-10:00, 15:00-18:00, 21:00-23:00 | 0.7 | 0.8 | **1.5** |
| ☀️ 峰时 | 10:00-15:00, 18:00-21:00 | 1.0 | 0.8 | **1.8** |

### 4.2 时间加速模拟

为加速测试验证，系统实现了 60 倍时间加速：

```python
SPEED_MULTIPLIER = 60  # 5秒真实时间 = 5分钟模拟时间

def scaled_elapsed_hours(start_time, now=None):
    elapsed_seconds = (now - start_time).total_seconds()
    return max(elapsed_seconds / 3600 * SPEED_MULTIPLIER, 0)
```

充电进度实时计算，前端每 1 秒刷新进度条，用户可直观看到加速充电过程。

### 4.3 自动调度算法

`run_scheduler()` 是核心调度函数，执行以下逻辑：

1. **结算已完成充电** — 检查充电中车辆是否达到目标电量，自动结算
2. **推进已分配队列** — 充电桩空闲时，自动从队列中启动下一辆车
3. **等候区分配** — 将等候区车辆按最小负载策略分配到充电桩
4. **故障等待处理** — 处理因故障等待的车辆，重新分配可用桩

**负载均衡策略**：选择当前负载（充电中+排队中）最小的充电桩进行分配。

### 4.4 故障调度策略

系统支持 **3 种故障调度策略**：

| 策略 | 说明 | 调度逻辑 |
|:----:|:----:|---------|
| **sequence** | 时间顺序调度 | 按等待时间从早到晚重新分配 |
| **priority** | 故障队列优先 | 故障等待车辆优先于普通等候车辆 |
| **shortest** | 最短时长调度 | 选能最快完成充电的桩（耗时最短） |

### 4.5 暂停超时罚金

```python
PAUSE_TIMEOUT_MINUTES = 10  # 暂停超时10分钟
PAUSE_PENALTY = 5.0         # 罚金¥5
```

定时检查 `enforce_pause_timeout()` 扫描所有暂停中的车辆，超过 10 分钟自动：
- 强制结束充电并生成账单
- 追加 ¥5 罚金到账单备注
- 用户累计警告次数 `warning_count` +1

## 六、实验总结与心得

### 6.1 项目成果

本实验成功设计并实现了一套完整的智能充电桩调度计费系统，具备以下核心能力：

1. **完整的用户管理**：支持用户注册/登录、充电请求提交/修改/取消、实时进度查看
2. **智能调度算法**：自动将等候车辆分配至最优充电桩，支持负载均衡
3. **分时电价计费**：峰平谷三时段电价 + 服务费，账单准确无误
4. **故障处理机制**：三种故障调度策略，支持车辆自动重分配
5. **暂停超时管控**：10 分钟超时自动结束并追加罚金
6. **管理监控面板**：充电桩监控、队列查看、统计报表、用户管理

### 6.2 技术亮点

- **单文件全栈应用**：后端逻辑集中在 `main.py`，前端集中在 `index.html`，结构清晰
- **60× 时间加速**：支持快速模拟充电过程，方便测试验证
- **自动化调度**：无需人工干预，系统自动完成从等候→排队→充电→结算的全流程
- **前端实时刷新**：进度每秒更新，暂停计时实时显示

### 6.3 改进方向

1. **WebSocket 实时推送**：当前使用前端轮询，可改为 WebSocket 推送提高实时性
2. **数据库持久化**：SQLite 适合开发测试，生产环境可迁移至 PostgreSQL
3. **密码安全增强**：注册 API 密码长度限制可考虑 6 位以上
4. **单元测试覆盖**：增加 pytest 单元测试，提高代码可靠性
5. **部署容器化**：提供 Dockerfile，支持一键部署
