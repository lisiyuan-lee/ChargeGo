## 2. 模块间调用关系总图

```
前端 (Vue3 SPA)
  ├── api/auth.js      ──→ AccountController   ──→ AccountService      (孙傲然)
  ├── api/request.js   ──→ RequestController   ──→ SchedulingService   (刘烨鑫)
  │                                           ──→ QueueManager         (冼炬恩)
  │                                           ──→ ChargingService      (肖航)
  ├── api/bill.js      ──→ BillController      ──→ BillingService       (刘烨鑫)
  └── api/admin.js     ──→ AdminController     ──→ ChargingPileService  (芦笑存)
                         ──→ MonitorController ──→ MonitoringService    (刘烨鑫)

WebSocket 实时推送:
  QueueWebSocketHandler ──→ /topic/queue/{userId}      (排队进度)
  QueueWebSocketHandler ──→ /topic/queue/{userId}      (叫号通知)
  PileWebSocketHandler  ──→ /topic/pile/{pileId}       (桩状态变更)
  PileWebSocketHandler  ──→ /topic/charging/{userId}   (充电进度)
  PileWebSocketHandler  ──→ /topic/admin/faults        (故障告警)

Redis 共享状态:
  queue:fast:waiting  (ZSet) — QueueManager 写, SchedulingService 读
  queue:slow:waiting  (ZSet) — QueueManager 写, SchedulingService 读
  pile:status:{pileId} (Hash) — MonitoringService 写, ChargingService 读
  session:token:{userId} (String) — AccountService 写, JwtAuthenticationFilter 读
```

## 3. 前后端 API 接口清单（联调核心）

### 3.1 统一基础约定

| 对齐项 | 约定内容 |
|--------|---------|
| 通信协议 | HTTP/1.1 (开发阶段)，未来可升 HTTPS |
| 数据格式 | JSON，Content-Type: application/json |
| 字符编码 | UTF-8 |
| 前端 Base URL | `/api`（Vite dev server 代理到 `localhost:8080`）|
| 后端端口 | `8080`（开发环境） |
| 统一响应格式 | `{"code": 0, "message": "success", "data": {...}}` |
| 鉴权方式 | JWT Token，Header: `Authorization: Bearer <token>` |
| 请求超时 | 前端 Axios 15s，后端建议 10s |
| 跨域配置 | CorsConfig.java 已配置，允许 `localhost:5173` |

### 3.2 账户模块接口（孙傲然 ↔ 前端）

| 接口 | 方法 | 路径 | 请求体 | 响应 data | 负责人 |

**联调检查点**:
- [ ] RegisterRequest 字段与前端表单字段一致

## 4. Service 层接口依赖（后端模块间联调）

### 4.1 SchedulingService（刘烨鑫）接口契约

```java
void enqueue(ChargingRequest request);           // 请求入队
void dequeue(Long requestId, String chargeMode); // 请求出队
ChargingRequest scheduleNext(String chargeMode);  // 调度下一辆
void rescheduleFaultedRequest(ChargingRequest request); // 故障再调度
int getQueuePosition(Long requestId, String chargeMode);
int getQueueLength(String chargeMode);
```

**调用方**: RequestController（冼炬恩）、MonitoringService（刘烨鑫）

### 4.2 QueueManager（冼炬恩）接口契约

```java
void addToQueue(String queueKey, Long requestId, double score);
void removeFromQueue(String queueKey, Long requestId);
Long peekFirst(String queueKey);
long getQueueSize(String queueKey);
long getRank(String queueKey, Long requestId);
```

**Redis Key 常量**:
- `queue:fast:waiting` → FAST_QUEUE_KEY
- `queue:slow:waiting` → SLOW_QUEUE_KEY

**ZSet score 规则**: `score = estimatedDuration`（预计充电时长，秒），值越小优先级越高（SJF 调度）

**调用方**: SchedulingServiceImpl（刘烨鑫）读写队列

### 4.3 ChargingService（肖航）接口契约

```java
void startCharging(Long requestId, Long pileId);
ChargingStateDTO getState(String carId);
void endCharging(Long sessionId);
```

**调用方**: RequestController（冼炬恩）

**联调关键**: 
- `startCharging` 调用后需更新 `ChargingPile.status = CHARGING`，`ChargingPile.currentSessionId`
- `endCharging` 调用后触发 `BillingService.generateBill(session)` 生成账单

### 4.4 BillingService（刘烨鑫）接口契约

```java
Bill generateBill(ChargingSession session);
List<BillSummaryDTO> listByUser(Long carId, String date);
BillDetailDTO getDetail(Long billId);
```

**调用方**: ChargingService（肖航）在充电结束时调用 `generateBill`；BillController（刘烨鑫）调用查询方法

**generateBill 内部逻辑**（2026-06-15 设计确认）:
1. 调用 `TimeSegmentFeeStrategy.splitByBoundaries(startTime, endTime)` 按时段边界切分充电区间
2. 按时间比例分摊电量到峰/平/谷各段，计算分段费用
3. 组装 Bill 实体（含 6 个分段字段 + 汇总字段），`payStatus = UNPAID`
4. `BillDAO.insert(bill)`

### 4.5 MonitoringService（刘烨鑫）接口契约

```java
PileStateDTO getPileState(Long pileId);
List<PileStateDTO> getAllPileStates();
void detectFaults();
List<FaultRecord> getActiveFaults();
void resolveFault(Long faultId);
```

**调用方**: MonitorController（刘烨鑫）、SchedulingService（刘烨鑫，故障时触发再调度）

### 4.6 AccountService（孙傲然）接口契约

```java
void register(RegisterRequest request);
void setPassword(Long userId, String newPassword);
LoginResponse login(LoginRequest request);
```

**调用方**: AccountController（孙傲然）

## 6. 状态枚举统一表

### 6.1 充电桩状态 (ChargingPile.status)

| 枚举值 | 含义 | 对应概念设计 state | 触发方 |
|--------|------|--------------------|--------|
| IDLE | 空闲可用 | RUNNING | AdminController powerOn / startPile |
| CHARGING | 正在充电 | — | ChargingService.startCharging |
| FAULT | 故障 | FAULT | MonitoringService.detectFaults |
| OFFLINE | 离线/关机 | OFF | AdminController powerOff |

### 6.2 充电请求状态 (ChargingRequest.status)

| 枚举值 | 含义 | 触发条件 | 设置方 |
|--------|------|---------|--------|
| WAITING | 排队等待 | 提交请求后 | RequestController / QueueManager |
| CHARGING | 充电中 | 分配到桩后 | SchedulingService / ChargingService |
| COMPLETED | 已完成 | 充电结束 | ChargingService |
| CANCELLED | 已取消 | 用户取消 | RequestController |

### 6.3 充电会话状态 (ChargingSession.status)

| 枚举值 | 含义 | 触发条件 | 设置方 |
|--------|------|---------|--------|
| CHARGING | 充电中 | 启动充电后 | ChargingService |
| COMPLETED | 正常完成 | 充电结束 | ChargingService |
| INTERRUPTED | 故障中断 | 故障检测到 | MonitoringService |

### 6.4 计费时段定义

> **注**（2026-06-15）: `Bill.timeSegment` 字段已废弃。时段信息现由分时段字段（`peak_kwh`/`flat_kwh`/`valley_kwh`）隐含表达。以下时段边界仅用于 `TimeSegmentFeeStrategy.splitByBoundaries()` 切分算法。

| 常量 | 含义 | 时段 | 电价 (元/kWh) |
|------|------|------|---------------|
| PEAK | 峰时 | 08:00-12:00, 17:00-22:00 | 1.20 |
| FLAT | 平时 | 12:00-17:00 | 0.80 |
| VALLEY | 谷时 | 22:00-08:00 | 0.40 |

跨时段充电时，按各时段实际时长比例分摊电量，分钟级精度。

### 6.5 用户角色 (User.role)

| 枚举值 | 含义 |
|--------|------|
| USER | 普通用户 |
| ADMIN | 管理员 |

### 6.6 故障类型 (FaultRecord.faultType)

| 枚举值 | 含义 |
|--------|------|

## 11. 分时计费参数对齐

计费参数由 `AdminController.setParameters` 配置，BillingService 使用：

| 参数 | 说明 | 建议默认值 |
|------|------|-----------|
| peakPrice | 峰时电价 (元/kWh) | 1.20 |
| flatPrice | 平时电价 (元/kWh) | 0.80 |
| valleyPrice | 谷时电价 (元/kWh) | 0.40 |
| serviceFeeRate | 服务费率 (元/kWh) | 0.50 |
| peakStart1 / peakEnd1 | 第一峰时 | 08:00 / 12:00 |
| peakStart2 / peakEnd2 | 第二峰时 | 17:00 / 22:00 |
| flatStart / flatEnd | 平时 | 12:00 / 17:00 |

**计费公式**（2026-06-15 修正）:
```
跨时段切分: 以 08:00/12:00/17:00/22:00 为边界将充电区间切为若干段
每段电量 = totalKwh × (该段时长 / 总时长)
每段费用 = 每段电量 × 该时段电价

peak_fee  = peak_kwh × 1.20
flat_fee  = flat_kwh × 0.80
valley_fee = valley_kwh × 0.40
charging_fee = peak_fee + flat_fee + valley_fee
service_fee = charge_amount × 0.50
total_amount = charging_fee + service_fee
```

**跨时段计算**: 按时间比例分配电量，精度到分钟。详单 DTO 展示各时段用电量和费用明细。
